رابط برنامه نویسی کاربردی (API) راهی برای ارتباط دو یا چند برنامه کامپیوتری با یکدیگر است. این یک نوع رابط نرم افزاری است که خدماتی را به سایر قطعات نرم افزار ارائه می دهد.[1] سند یا استانداردی که نحوه ساخت یا استفاده از چنین اتصال یا رابطی را توضیح می دهد، مشخصات API نامیده می شود. سیستم رایانه ای که این استاندارد را برآورده می کند گفته می شود که یک API را پیاده سازی یا افشا می کند. اصطلاح API ممکن است به مشخصات یا پیاده سازی اشاره داشته باشد. در حالی که رابط کاربری یک سیستم نحوه تعامل کاربران نهایی با سیستم مورد نظر را دیکته می کند، API آن دیکته می کند که چگونه کدی بنویسد که از قابلیت های آن سیستم استفاده کند.
بر خلاف یک رابط کاربری که کامپیوتر را به یک شخص متصل می کند، یک رابط برنامه نویسی کاربردی کامپیوترها یا قطعات نرم افزار را به یکدیگر متصل می کند. در نظر گرفته نشده است که مستقیماً توسط شخصی (کاربر نهایی) غیر از برنامه نویس رایانه ای که آن را در نرم افزار ادغام می کند استفاده شود. یک API اغلب از قسمت های مختلفی تشکیل شده است که به عنوان ابزار یا خدماتی عمل می کنند که در دسترس برنامه نویس هستند. گفته می شود برنامه یا برنامه نویسی که از یکی از این قسمت ها استفاده می کند آن بخش از API را فراخوانی می کند. فراخوانی هایی که API را تشکیل می دهند به عنوان زیر روال، متد، درخواست یا نقطه پایانی نیز شناخته می شوند. مشخصات API این فراخوان ها را تعریف می کند، به این معنی که نحوه استفاده یا پیاده سازی آنها را توضیح می دهد.
