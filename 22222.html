اغلب کاربران می پرسند  headless WordPress ( وردپرس بدون هد ) چیست و آیا باید از آن استفاده کنند؟

هدلس وردپرس ، ناحیه مدیریت بک‌اند وردپرس را از قسمت فرانت اند که بازدیدکنندگان مشاهده می کنند جدا می‌کند. این امکان را به توسعه‌دهندگان می‌دهد که از فناوری‌های مختلف برای فرانت اند یک وب‌سایت استفاده کنند و در عین حال ثبات و امنیت وردپرس را نیز داشته باشند.

در این مقاله توضیح خواهیم داد که هدلس وردپرس چیست و آیا باید از آن استفاده کنید. ما همچنین در مورد مزایا و معایب یک وب سایت هدلس وردپرس به تفصیل صحبت خواهیم کرد.

در اینجا مروری سریع از موضوعاتی است که در این راهنما پوشش خواهیم داد:

  • آیا باید از یک وب سایت هدلس وردپرس استفاده کنید؟
 

هدلس وردپرس چیست؟

هدلس وردپرس یک اصطلاح برای هر وب سایتی است که از وردپرس به عنوان نرم افزار بک اند برای مدیریت داده ها و محتوا استفاده می کند، اما از فناوری های جداگانه برای نمایش داده ها در فرانت اند استفاده می کند.

Back-end اصطلاحی است که برای بخش مدیریت یک وب سایت استفاده می شود. اینجا جایی است که محتوایی مانند پست ها و صفحات ایجاد می شوند، افزونه ها نصب می شوند، نظرات مدیریت می شوند و موارد دیگر.

عکس

ناحیه Front-end برای وب‌سایت شما، نقطه دید قابل مشاهده عموم است؛ محیطی که کاربران هنگام بازدید از وب‌سایت در مرورگرهای خود مشاهده می‌کنند.

عکس

 

وردپرس معمولاً هر دو جنبه را در یک نرم افزار قدرتمند برای شما فراهم می کند. شما می توانید به سادگی از هر قالب وردپرس برای نمایش وب سایت خود استفاده کنید.

با این حال، برخی از توسعه‌دهندگان و صاحبان وب‌سایت‌ها ممکن است نیاز به استفاده از فناوری‌های دیگری مانند React.js، Angular.js، Faust.js و سایرین برای ایجاد یک فرانت‌اند سفارشی برای وب‌سایت خود داشته باشند.

هدلس وردپرس این امکان را فراهم می کند، تا از فناوری‌های بالا استفاده کنید، در حالی که همچنان از وردپرس در بک اند برای مدیریت آسان‌تر محتوا استفاده می‌کنید.

هدلس وردپرس چگونه کار می کند؟

وردپرس دارای یک منطقه مدیریتی با کاربری آسان است که در آن محتوا ایجاد می کنید و وب سایت خود را مدیریت می کنید. همچنین شامل یک موتور قالب برای نمایش قسمت فرانت اند سایت شما با استفاده از تم ها می باشد.

وردپرس همچنین با یک API به نام WordPress REST API عرضه می شود.

این API یک رابط برنامه نویسی برای توسعه دهندگان فراهم می کند تا به داده های وب سایت وردپرس در قالب JSON دسترسی داشته باشند.

هنگام ساخت یک وب سایت هدلس وردپرس ، توسعه دهندگان از REST API برای واکشی داده ها از وردپرس استفاده می کنند. پس از آن، آنها می‌توانند از آن در برنامه‌های خود یا سایر فناوری‌های فرانت‌اند مانند React.js، Angular.js، Faust.js و موارد دیگر برای ایجاد یک وب‌سایت سفارشی استفاده کنند.

مزایای استفاده از Headless WordPress چیست؟

هدلس وردپرس می تواند در برخی سناریوهای خاص مفید باشد. ما در مورد برخی از مزایای استفاده از وردپرس بدون هد در آن صحبت خواهیم کرد.

  1. سرعت بالاتر

توسعه دهندگان بر این باورند که وردپرس بدون هد بسیار سریعتر از یک وب سایت وردپرس معمولی است.

با جدا کردن فرانت اند با استفاده از فناوری هایی مانند React.js، بسیاری از توسعه دهندگان سعی می کنند کد کارآمدتری نسبت به اکثر قالب های وردپرس ارائه کنند.

با این حال، تجربه نشان می‌دهد که تکنیک‌های ساده بهینه‌سازی عملکرد WordPress برای اکثر وب‌سایت‌های کسب‌وکار کوچک همانند headless به خوبی کار می‌کند.

  1. ادغام داده ها در برنامه های دیگر:

اگر به دنبال ادغام بخش‌هایی از وب‌سایت WordPress خود، مانند وبلاگ یا بخش‌های دیگر، در یک برنامه موبایل هستید، Headless WordPress  می‌تواند به شما کمک کند. شما می‌توانید داده‌های WordPress را به برنامه‌های دیگر منتقل کرده و از فناوری‌های مختلف برای ایجاد تجربه کاربری یکپارچه استفاده نمایید.

به عنوان مثال، اگر یک وب‌سایت استاتیک دارید و فقط برای بخش خاصی مانند وبلاگ، نظرات یا فروشگاه آنلاین به بخش پویا نیاز دارید ، Headless WordPress می‌تواند داده‌های وردپرس را در وب‌سایت استاتیک شما ادغام نماید.

  1. انتشار در پلتفرم‌های مختلف:

هدلس وردپرس ، داده‌های شما را قابل حمل می‌کند و امکان انتقال آنها به هر نقطه‌ای را فراهم می‌کند. شما می‌توانید از داده‌ها در کمپین‌های بازاریابی، ارسال پست در پلتفرم‌های رسانه‌های اجتماعی، استفاده در دستگاه‌های IoT مانند بلندگوهای هوشمند، تغذیه ابزارهای هوش مصنوعی، اتصال به دستیاران مجازی و موارد دیگر استفاده کنید.

  1. هدلس وردپرس بسیار مقرون به صرفه تر از CMS هدلس اختصاصی است

هدلس وردپرس به مراتب مقرون به صرفه تر و قدرتمندتر از راه حل های اختصاصی "CMS سازمانی" مانند Contentful

 است و با استفاده از آن می‌توانید هزینه‌های میزبانی را به طور قابل توجهی کاهش دهید (بدون محاسبه هزینه‌های توسعه).

معایب یک وب سایت هدلس وردپرس چیست؟

ایجاد یک وب سایت هدلس وردپرس نیز ممکن است معایبی برای وب سایت شما داشته باشد. بیایید نگاهی به چند مورد از آنها بیندازیم.

  1. هدلس وردپرس در مقایسه با سایت های وردپرس سنتی گران تر است

شما باید یک توسعه دهنده استخدام کنید تا یک ظاهر سفارشی برای وب سایت خود ایجاد کنید. این به راحتی می تواند به محدوده ده ها هزار دلار برسد.

 و اگر توسعه‌دهنده هستید، زمان با ارزشی را صرف ساختن یک ظاهر سفارشی خواهید کرد.

حتی اگر از یک فریمورک توسعه front-end استفاده می‌کنید، زمان و تلاش بیشتری نسبت به استفاده از یک تم آماده وردپرس با کش مناسب در سطح سرور / DNS نیاز دارد.

  1. برخی از پلاگین های وردپرس ممکن است کار نکنند

افزونه ها مانند برنامه هایی برای وب سایت وردپرس شما هستند و به شما امکان می دهند ویژگی های جدیدی را به سایت خود اضافه کنید و عملکرد آن را گسترش دهید.

یک وب سایت هدلس وردپرس ممکن است نتواند از برخی از پلاگین های وردپرس استفاده کند. به عنوان مثال، برخی از داده‌های پلاگین وردپرس ممکن است با استفاده از REST API در دسترس نباشند، یا ویژگی‌های پلاگین ممکن است برای عملکرد صحیح به یک تم وردپرس نیاز داشته باشند.

با این اوصاف، پلاگین های محبوب وردپرس مانند AIOSEO دارای یک پسوند REST API هستند که به شما امکان می‌دهد سایت‌های هدلس وردپرس با سئو ایجاد کنید.

  1. سردردهای نگهداری

شما اساسا از پلتفرم های جداگانه برای قسمت فرانت اند و بک اند سایت خود استفاده خواهید کرد و باید هر دو را جداگانه نگهداری کنید. گاهی اوقات، ممکن است همه چیز از هر طرف خراب شود. برای حل این مشکلات باید به توسعه دهندگان اعتماد کنید. یا وقت ارزشمند خود را صرف تعمیر آنها خواهید کرد. پیچیدگی اضافی با رشد کسب و کار شما، کار را سخت تر می کند.

آیا باید از یک وب سایت وردپرس بدون سر استفاده کنید؟

به نظر ما، اکثر کاربران وردپرس نیازی به استفاده از وب سایت هدلس وردپرس یا انواع دیگر فریم ورک بدون هد ندارند.

 گران است، پیچیده است و برای اکثریت قریب به اتفاق صاحبان وب سایت، وبلاگ نویسان، مشاغل کوچک یا فروشگاه های آنلاین مناسب نیست.

 به عنوان مثال، اگر سرعت و عملکرد نگرانی اصلی شما باشد، وب سایت وردپرس موجود شما به راحتی می تواند هر وب سایت هدلس وردپرس را شکست دهد. تمام پلاگین های وردپرس برتر دارای ویژگی های کش صفحه، پیش بارگذاری کش، کش اشیاء و فشرده سازی gzip هستند.

روشن کردن این ویژگی‌ها می‌تواند به طور قابل توجهی سرعت وب‌سایت شما را بدون استخدام هیچ توسعه‌دهنده‌ای افزایش دهد. آن را با یک CDN قدرتمند و فایروال برنامه وب مانند Cloudflare یا Sucuri ترکیب کنید، وب‌سایت شما حتی سریعتر خواهد بود.

مطالعه موردی را ببینید که چگونه WPBeginner را سریع‌تر از وب‌سایت‌های بدون سر بارگذاری کردیم. در مورد ادغام ها، بیش از 59000 افزونه وردپرس وجود دارد که تقریباً با هر پلتفرم شخص ثالث موجود ادغام می شود. حتی ابزارهایی مانند Uncanny Automator وجود دارد که امکان ادغام با وب سایت های وردپرس و ابزارهای شخص ثالث را با استفاده از Zapier فراهم می کند.

 اگر به یک ادغام نیاز دارید که از طریق افزونه وردپرس موجود در دسترس نباشد، شاید باید یک توسعه دهنده را برای نوشتن یک پلاگین سفارشی استخدام کنید. این هزینه بسیار کمتری خواهد داشت و حتی می توانید با فروش آن به عنوان دانلود دیجیتال به سایر کاربران وردپرس، به صورت آنلاین درآمد کسب کنید.

چه کسی باید از هدلس وردپرس استفاده کند؟

ما معتقدیم که هدلس وردپرس فقط باید در سناریوهای بسیار خاص استفاده شود. ابتدا، باید ارزیابی کنید که آیا واقعاً نیازی به CMS بدون هد وجود دارد یا خیر.

به عنوان مثال، شرکتی که یک وب سایت یا یک برنامه سفارشی ساخته است، ممکن است برای ویژگی‌های مدیریت محتوای آسان‌تر به وردپرس نیاز داشته باشند، و در این سناریو استفاده از وردپرس بدون هد منطقی است.

 ثانیا، یا باید بودجه لازم برای استخدام و حفظ توسعه دهندگان را داشته باشید یا مهارت لازم برای حفظ یک CMS بدون هد را داشته باشید.

 به یاد داشته باشید، یک CMS وردپرس بدون هد برای ایجاد یک برنامه سفارشی جدید یا برنامه ای که از وردپرس برای ایجاد و مدیریت آسان تر محتوا استفاده می کند، استفاده می شود.

نکته مهمی که باید با وردپرس Headless مراقب آن باشید

به‌عنوان یک صاحب کسب‌وکار غیر فنی، گاهی اوقات سخت است که بدانید آیا واقعاً به هدلس وردپرس نیاز دارید یا خیر. برخی از توسعه‌دهندگان و آژانس‌های وردپرس سعی می‌کنند وب‌سایت‌های هدلس را به شما بفروشند، و شما باید در مورد وضعیت خود آگاه باشید و به یاد داشته باشید که انگیزه آنها این است که از شما پول بیشتری بگیرند.

اگر تنها دلیلی که آنها وردپرس بدون هد را توصیه می‌کنند سرعت یا بهینه‌سازی برای موتورهای جستجو است، باید از آنها بپرسید که تا به حال در رابطه با بهینه‌سازی سرعت وب‌سایت شما چه اقداماتی انجام داده‌اند و چقدر تغییر به وردپرس هدلس می‌تواند تأثیرگذار باشد. اگر پاسخ روشنی دریافت نکردید، باید رویکرد دیگری را جستجو کنید.

تعجب خواهید کرد که برای اکثریت قریب به اتفاق وب سایت ها، پیشرفت ها ناچیز است، اما هزینه توسعه و نگهداری وب سایت های وردپرس بدون هد بسیار بالاتر است.

هر چیزی که انجام می‌دهید، قطعا اجازه ندهید کسی شما را متقاعد کند که از وردپرس به چیزی مانند Contentful تغییر دهید، زیرا بسیار ناامید خواهید شد. شنیده‌ایم که بسیاری از صاحبان کسب‌وکار تنها به اشتباه متوجه شدند و تغییر می‌دهند، سپس دوباره به وردپرس بازمی‌گردند.
